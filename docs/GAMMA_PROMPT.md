# 🎨 برومبت Gamma - نظام إدارة المقاولات

## 📋 البرومبت الكامل

```
أنشئ عرض تقديمي احترافي باللغة العربية عن "نظام إدارة المقاولات - المستقبل لإدارة المقاولات" 
لمشروع تخرج في قسم الهندسة المعلوماتية.

المتطلبات:
1. التصميم: استخدم قالب احترافي بألوان متسقة (Primary: #3A424F رمادي داكن، Secondary: #4A5568، Accent: #5A6578)
2. الخطوط: استخدم خطوط واضحة للعربية (Cairo أو Arial) والإنجليزية (Times New Roman)
3. الرسوم: أضف رسوم بيانية ومخططات تلقائياً حيثما كان ذلك مناسباً
4. الصور: استخدم أيقونات ورسوم توضيحية مناسبة
5. عدد الشرائح: 16 شريحة

المحتوى:

الشريحة 1 - صفحة العنوان:
العنوان: نظام إدارة المقاولات
العنوان الفرعي: المستقبل لإدارة المقاولات
Construction Management System
مشروع تخرج - قسم الهندسة المعلوماتية - جامعة الجزيرة الخاصة
العام الدراسي: 2025-2026

الشريحة 2 - نظرة عامة:
العنوان: نظرة عامة على المشروع
المحتوى:
• نظام شامل لإدارة المشاريع البنائية
• يربط بين المقاولين والعملاء
• إدارة متكاملة للمواد والمدفوعات والمشاريع
الهدف: تحويل إدارة المقاولات من الطرق التقليدية إلى النظام الرقمي
أضف رسم بياني يوضح الفرق بين الطريقة التقليدية والرقمية

الشريحة 3 - المشكلة:
العنوان: التحديات في قطاع المقاولات
المحتوى:
❌ استخدام طرق تقليدية (أوراق، Excel)
❌ صعوبة تتبع المشاريع والمواد
❌ عدم وجود تواصل فعال
❌ صعوبة إدارة المدفوعات
❌ عدم وجود تقارير دقيقة
أضف مخطط دائري يوضح نسبة المشاكل

الشريحة 4 - الحل:
العنوان: نظام إدارة المقاولات الرقمي
المحتوى:
✅ إدارة شاملة (المشاريع، المواد، المدفوعات)
✅ تقارير تفاعلية (مالية، مشاريع، مواد)
✅ واجهة سهلة الاستخدام (بديهية، متجاوبة، حديثة)
أضف مخطط مقارنة Before/After

الشريحة 5 - الميزات الرئيسية:
العنوان: الميزات الرئيسية
قسم الشريحة إلى عمودين:
العمود الأول - للمقاول:
• لوحة تحكم شاملة
• إدارة المشاريع
• إدارة المواد والموردين
• إدارة المدفوعات
• التقارير المالية

العمود الثاني - للعميل:
• متابعة المشاريع
• إرسال الطلبات
• التواصل المباشر
• عرض التقارير
أضف أيقونات لكل ميزة

الشريحة 6 - التقنيات المستخدمة:
العنوان: التقنيات والأدوات
قسم إلى 4 أقسام:
Frontend: React.js 19.2.0, React Router DOM, CSS3
Backend: Node.js, Express.js 4.18.2, MongoDB 8.0.3, Mongoose
الأمان: JWT, bcryptjs, Google OAuth 2.0
النشر: Vercel, Render, MongoDB Atlas
أضف شعارات التقنيات (استخدم أيقونات)

الشريحة 7 - البنية المعمارية:
العنوان: البنية المعمارية للنظام
أنشئ مخطط تدفق يوضح:
Frontend (React.js) على Vercel
    ↓ HTTPS/REST API
Backend (Node.js/Express) على Render
    ↓ MongoDB Connection
Database (MongoDB Atlas)
أضف رسم تخطيطي للبنية المعمارية

الشريحة 8 - قاعدة البيانات:
العنوان: تصميم قاعدة البيانات
المحتوى: 10 Models رئيسية
1. User - المستخدمون
2. Project - المشاريع
3. Material - المواد
4. Supplier - الموردين
5. Purchase - المشتريات
6. Payment - المدفوعات
7. Issue - إصدار المواد
8. Contract - العقود
9. Request - الطلبات
10. Report - التقارير
أضف مخطط ERD مبسط يوضح العلاقات

الشريحة 9 - الواجهات الرئيسية:
العنوان: واجهات المستخدم
قسم إلى قسمين:
صفحات المقاول: لوحة التحكم، إدارة المشاريع، إدارة المواد، إدارة المدفوعات، التقارير
صفحات العميل: لوحة التحكم، المشاريع، إرسال طلب، التقارير
أضف أيقونات للصفحات

الشريحة 10 - الأمان:
العنوان: نظام الأمان
المحتوى:
✅ تسجيل دخول آمن (كلمات مرور مشفرة، JWT Tokens, Google OAuth)
✅ حماية البيانات (HTTPS، حماية API، التحقق من الصلاحيات)
✅ الأمان على مستوى قاعدة البيانات
أضف رسم تخطيطي يوضح طبقات الأمان

الشريحة 11 - التكامل مع الخدمات:
العنوان: التكامل مع الخدمات الخارجية
قسم إلى قسمين:
Google OAuth: تسجيل دخول سريع وآمن، لا حاجة لكلمة مرور
Stripe Payment: دفع إلكتروني آمن، دعم البطاقات، فواتير تلقائية
أضف شعارات Google و Stripe

الشريحة 12 - النتائج:
العنوان: ما تم إنجازه
المحتوى:
✅ نظام متكامل يعمل بالكامل
✅ واجهة مستخدم حديثة ومتجاوبة
✅ قاعدة بيانات منظمة (10 Models)
✅ نظام أمان قوي
✅ نشر على السحابة (3 منصات)
أضف مخطط أعمدة يوضح الإنجازات

الشريحة 13 - الإحصائيات:
العنوان: إحصائيات المشروع
المحتوى:
📊 10 Models في قاعدة البيانات
📄 15+ صفحة في الواجهة
🔌 20+ API Endpoint
📱 100% متجاوب مع جميع الأجهزة
⚡ 99.9% وقت تشغيل
🔒 نظام أمان متعدد الطبقات
🌐 نشر على 3 منصات سحابية
أضف مخطط دائري أو أعمدة للأرقام

الشريحة 14 - التحديات والحلول:
العنوان: التحديات والحلول
قسم إلى عمودين:
التحديات:
• تعقيد إدارة الحالة في React
• التكامل مع Stripe Payment
• الأمان وحماية البيانات

الحلول:
• استخدام React Hooks
• وثائق Stripe الشاملة
• JWT + bcrypt + Google OAuth
أضف مخطط مقارنة

الشريحة 15 - المستقبل:
العنوان: التحسينات المستقبلية
المحتوى:
📱 تطبيق جوال (iOS و Android)
🤖 ذكاء اصطناعي (التنبؤ بالتكاليف)
📊 تخطيط المشاريع المتقدم (Gantt Charts)
👥 إدارة الموارد البشرية
أضف مخطط زمني للخطط المستقبلية

الشريحة 16 - الخلاصة:
العنوان: الخلاصة
المحتوى:
• نظام شامل ومتكامل لإدارة المقاولات
• استخدام تقنيات حديثة ومتطورة
• واجهة مستخدم سهلة ومتجاوبة
• جاهز للاستخدام الفعلي
النتيجة: نظام متكامل يساهم في التحول الرقمي لقطاع المقاولات

في الأسفل: شكراً لكم - Thank You - أسئلة؟

التنسيق المطلوب:
- العناوين: Bold، 32-36 نقطة
- النصوص: 18-24 نقطة
- Bullet Points: واضحة ومتسقة
- المسافات: مناسبة
- الألوان: متسقة في جميع الشرائح
- الرسوم: احترافية وواضحة
- الصور: عالية الجودة

أضف تلقائياً:
- رسوم بيانية حيثما كان ذلك مناسباً
- مخططات تدفق للعمليات
- أيقونات للعناصر الرئيسية
- رسوم توضيحية للتقنيات
- مخططات مقارنة
- مخططات إحصائية
```
